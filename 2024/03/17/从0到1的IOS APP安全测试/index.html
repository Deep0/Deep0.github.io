从0到1的IOS APP安全测试

作者：Deep

0x01前言

Android APP X 企业版加壳加固，代码高度混淆，各种防调试。

IOS APP ✓ Apple官方加壳，市场上加固混淆没有特别成熟方案，由于APP store审核严格，若加固混淆则很可能无法通过审核上架应用市场。

本文章主要介绍IPhone越狱，常用的越狱插件，砸壳，反编译。

0x02 越狱 Jailbreak

确定当前IOS 版本和设备型号，根据不同版本和设备型号使用不同的越狱工具。

越狱工具 限制OS版本 限制设备型号 工具链接 平台
checkra1n iOS12.0 - IOS14.5 devices (A7-A11) https://checkra.in/ MAC/Linux
unc0ver iOS 11.0 - 14.8 devices (A7-A11) https://unc0ver.dev/ Windows/iOS
palera1n iOS ≥15.0+ devices (A8-A11) https://palera.in/ MAC/Linux
Dopamine iOS ≥15.0+ devices (A9-A16) https://ellekit.space/dopamine/ iOS

此处以使用的为IPhone 8 Plus IOS 13.4，checkra1n越狱为例。

通过访问https://checkra.in/，下载安装checkra1n软件，将IPhone手机USB连接电脑，iPhone屏幕解锁状态下运行checkra1n。

![](./2024/03/17/从0到1的IOS APP安全测试/image_KYdmp9XhWv.png)

点击Start 开始进行越狱，继续点击Next进入到以下界面，点击Start后依照图示，同时按住电源键和音量减键，按住5秒后松开电源键并保持音量减键10秒，进入DFU模式。

![](./2024/03/17/从0到1的IOS APP安全测试/image_3ZgWG6cdCi.png)

接下来checkra1n会对iPhone进行越狱操作，越狱成功后手机自动重启桌面上会自动添加Cydia应用。

![](./2024/03/17/从0到1的IOS APP安全测试/image_OGj0XSjeLF.png)

0x03 越狱插件

越狱插件是在越狱后的iOS设备上安装的软件，越狱插件的功能非常多样，包括但不限于：改变系统界面和图标、增强应用的功能，如微信和QQ的防撤回、抢红包等功能，本节主要介绍做渗透中常用的越狱插件。

Cydia ： 越狱插件商店，在APP里可以添加下载源仓库链接并下载各种越狱插件。

AppStorePlus ： 越狱插件，可安装指定版本APP。

CrackerXI+ ： 越狱插件，砸壳工具，可解密官方加密的APP。

DumpDecrypter ： 越狱插件，砸壳工具，可解密官方加密的APP。

KernBypass ： 越狱插件，从内核hook绕过APP越狱检测机制。

Shadow ： 越狱插件，Hook相关方法绕过APP越狱检测机制。

iCleaner ： 越狱插件，清除系统中的缓存垃圾文件。

SSL KIll Switch 2 /3： 越狱插件，绕过SSL证书绑定，禁止SSL证书验证。

Frida ：越狱插件，用来调试 hook APP。

0x04 SSH连接越狱iPhone

SSH连接越狱iPhone可以更方便快捷得与iPhone进行调试交互，Cydia商店提供了OpenSSH的教程，进入主界面后点OpenSSH访问教程即可进入。

![](./2024/03/17/从0到1的IOS APP安全测试/image_dEK4sFosw2.png)

点击蓝色字体OpenSSH，安装OpenSSH插件，安装完成后就可以SSH登录iPhone。

连接SSH有两种方式，第一种是通过iPhone与连接机器处于同一局域网下，可使用SSH连接iPhone机器的局域网IP。

![](./2024/03/17/从0到1的IOS APP安全测试/image_gtLABabnHB.png)

第二种方式是电脑通过USB / Type C / Lighting数据线连接iPhone，通过iproxy转发22端口到2222，然后再开一个终端窗口SSH连接本地IP端口2222，从而登录到iPhone命令终端。

![](./2024/03/17/从0到1的IOS APP安全测试/image_hiRW0g2aFz.png)

SSH 连接方式推荐使用第二种，因为第一种方式需要获取iPhone的局域网IP，如果网络环境经常变动，每次获得的IP可能不一样，所以使用第二种方法更加方便快捷。

解决中文编码问题

IOS终端默认没有安装编码文件，导致中文无法再SSH终端输入和正常显示。

![](./2024/03/17/从0到1的IOS APP安全测试/image_saJ-V28zfx.png)

Mac端和iOS端的编码配置文件兼容，将Mac下的en_US.UTF-8编码文件传到iPhone。

![](./2024/03/17/从0到1的IOS APP安全测试/image_KFrSM6CvSf.png)

重新SSH进入iPhone，发现能够正常输入和显示中文。

![](./2024/03/17/从0到1的IOS APP安全测试/image_wLA-L-vKRO.png)

0x05 砸壳

砸壳指的是对已经加密的软件进行解密的过程，砸壳可以分为静态砸壳和动态砸壳两种方式。在渗透测试中，对APP进行逆向分析的前提就是需要先砸壳解密再进行分析。

动态砸壳

动态砸壳：从运行在进程内存空间中的可执行程序映像入手，通过转储内存内容来实现脱壳处理。

前提：APP能够在越狱设备正常运行，不闪退。

常用的工具：frida-ios-dump https://github.com/AloneMonkey/frida-ios-dump，CrackerXI+http://cydia.iphonecake.com

此处以CrackerXI+为例。

打开Cydia添加源http://cydia.iphonecake.com ，搜索CrackerXI+如下所示，进行安装。

![](./2024/03/17/从0到1的IOS APP安全测试/image_r9Ek4u3Kph.png)

在手机HOME界面找到CrackerXI+图标点击进入该插件，进入Seeings设置界面启用CrackerXI Hook功能。

![](./2024/03/17/从0到1的IOS APP安全测试/image_mTt84e462t.png)

![](./2024/03/17/从0到1的IOS APP安全测试/image_ZbWMAZyh9d.png)

在AppList页面点击对应APP，有两个选项，第一个是解密出完整的IPA文件（相当于Android的APK），第二个是只解密IPA里的二进制文件（APP的主程序）。

![](./2024/03/17/从0到1的IOS APP安全测试/image_HF7bBACWMF.png)

这里选择第一个选项，解密出完整的IPA文件。要解密的APP会自动打开，解密完成后会自动跳转回CrackerXI+界面并告知解密路径。

![](./2024/03/17/从0到1的IOS APP安全测试/image_A-4maqHsBo.png)

通过SSH连接iPhone通过图上描述的路径即可拿到解密后到IPA文件。

静态砸壳

静态砸壳：在了解应用的加密算法和逻辑后，不运行应用程序的情况下进行解密处理，APP存在越狱检测闪退可用这种方法。

常用的工具：Iridium （iOS 13.5 <-> 14.3） https://github.com/Lakr233/Iridium，appdecrypthttps://github.com/paradiseduo/appdecrypt

此处以appdecrypt为例。

下载appdecrypt： https://github.com/paradiseduo/appdecrypt/releases ，releases提供了ios、mac版本，也可以下载Source code进行编译，此处下载appdecrypt的ios版本。

![](./2024/03/17/从0到1的IOS APP安全测试/image_dffeQowzKF.png)

下载zip后解压出一个二进制文件，通过SSH连接iPhone上传到/tmp目录下。

![](./2024/03/17/从0到1的IOS APP安全测试/image_XXDsdMwB69.png)

![](./2024/03/17/从0到1的IOS APP安全测试/image_cJO9jyVxBt.png)

解密后的IPA包资源文件都被打包进dump.zip，将其提取到电脑端，解压即可看到，砸壳成功。

![](./2024/03/17/从0到1的IOS APP安全测试/image_8WUqSaT1Wa.png)

其他获取解密IPA的方式

除了动态解密和静态解密等手段外，还可以通过去网站上下载解密IPA文件，以下两个网站都可以下载解密IPA，但是并不能保证能够获得最新版本的解密IPA文件。

https://decrypt.day/

![](./2024/03/17/从0到1的IOS APP安全测试/image_rHFQJGSGtB.png)

https://armconverter.com/decryptedappstore/us

![](./2024/03/17/从0到1的IOS APP安全测试/image_tT2DtEnaKZ.png)

检测是否砸壳成功

获得解密IPA文件后，可通过对IPA包内对二进制文件执行otool命令查看是否加密。

![](./2024/03/17/从0到1的IOS APP安全测试/image_YyBApo3ZOM.png)

0x06反编译

反编译也是测试 iOS 应用程序的必经之路，只有经过反编译 APP 才能深入分析其内部结构、逻辑和代码。通过反编译，我们可以查看应用程序的源代码，研究其使用的算法和数据结构，寻找潜在的安全漏洞。

反编译IOS主要有两款软件，IDA Pro和Hopper。

IDA Pro，将解密二进制程序拖入IDA pro如图所示。

![](./2024/03/17/从0到1的IOS APP安全测试/image_vMsyFdn4su.png)

Hopper，将解密二进制程序拖入Hopper如图所示。

![](./2024/03/17/从0到1的IOS APP安全测试/image_Qf8A9cmmOU.png)

反编译后，可在伪代码中或字符串中查找硬编码密钥并加以利用。

![](./2024/03/17/从0到1的IOS APP安全测试/image_SRse3CJ5wc.png)

若APP传输过程中数据包被加密，可通过反编译出伪代码查找加密算法与加密密钥。

![](./2024/03/17/从0到1的IOS APP安全测试/image_f9bLMCgv2U.png)

0x07 代理抓包

IOS下的代理抓包设置和Android大同小异，常用的代理抓包方法分为两种：一是对网络设置代理，二是通过VPN连接方式进行抓包。

网络设置代理

进入 设置 - 无线局域网 - 选中加入的WI-FI网络 - 配置代理，在此界面输入代理服务器的IP地址和端口点击存储即可完成代理配置。

![](./2024/03/17/从0到1的IOS APP安全测试/image_WiU0z7gWCC.png)

![](./2024/03/17/从0到1的IOS APP安全测试/image_Tuk4QEYY5p.png)

VPN设置代理

有些APP不走系统代理，这时候可以通过设置VPN代理进行绕过。

安装Shadowrocket软件，添加节点，将节点类型设置为HTTP，并输入代理服务器的IP地址和端口，将全局路由选择代理选项，即可完成代理配置。

![](./2024/03/17/从0到1的IOS APP安全测试/image_erS3opdM91.png)

![](./2024/03/17/从0到1的IOS APP安全测试/image_-N5ZkO8zGR.png)

绕过SSL证书绑定

在一些APP中，可能存在证书固定等情况无法直接抓取HTTPS传输数据包，需要绕过才能抓HTTPS。

可通过安装SSL KIll Switch 2/3越狱插件或使用Objection的ios sslpinning disable功能，禁用APP内置的证书验证。

0x08 frida-trace 应用

前面叙述了如何一些渗透测试的技巧，那么在iOS端如何更好的去动态追踪一些参数呢？frida-trace就是可以进行动态跟踪，是Frida动态代码检测工具的一部分。

Frida是一种基于JavaScript的动态分析工具，可以用于逆向开发、应用程序的安全测试、反欺诈技术等领域。Frida主要用于在已安装的应用程序上运行自己的JavaScript代码，从而进行动态分析、调试、修改等操作，能够绕过应用程序的安全措施，可以助力于对应用程序进行逆向分析。

首先在电脑端安装frida工具，通过pip安装frida和frida-tools。

然后在iOS设备上的Cydia商店添加Frida软件源 https://build.frida.re ，然后在手机端安装Frida，frida的版本和电脑端的frida版本要保持一致，防止出现兼容性问题。

![](./2024/03/17/从0到1的IOS APP安全测试/image_ZXqc3GasiJ.png)

使用数据线将iOS设备与电脑连接，执行frida-ps命令，查看是否能够正常列举手机上的正在运行APP的列表，若能够正常输出列表则代表frida可正常使用。

![](./2024/03/17/从0到1的IOS APP安全测试/image_B23DxkaoQ7.png)

  • Frida-trace 通用逻辑
    • -m、-M等参数，支持通配符
      • *：表示任意个数的字符=所有的（类或方法名）

        • 举例

          -m “-[NSURL *]” // 匹配NSURL类的所有实例方法

          -m “+[NSURL *]” // 匹配NSURL类的所有类方法

          -m “ [NSURL ]” // 匹配NSURL类的所有方法

          -m “ [ URL *]” // 匹配以URL结尾类的所有方法

          -m “[URL *]” // 匹配以URL开头类的所有方法

          -m “ *[URL *]” // 匹配包含URL的类的所有方法

          -m “ *[URL login]” // 匹配包含URL的类的带login的所有方法

      • 注意
        • 不支持：?
          • 如果用了?，则会报错：Error: invalid query; format is: -[NS*Number foo:bar:], +[Foo foo*] or *[Bar baz]
    • 参数
      • -m
        • 语法：-m OBJC_METHOD, --include-objc-method OBJC_METHOD
          • include OBJC_METHOD
        • 含义：（要去hook的）包含的ObjC的（类和）方法
      • -M
        • 语法：-M OBJC_METHOD, --exclude-objc-method OBJC_METHOD
          • exclude OBJC_METHOD
        • 含义：（ 不要hook的）排除掉的ObjC的（类和）方法
      • -O
        • 语法：-O FILE, --options-file FILE
          • text file containing additional command line options
        • 含义：用Option文件，包含额外的命令行参数
          • 典型用法是，觉得-m、-M等要去hook和要去排除掉的类和方法，太多了，就转去放到Option文件中

frida-trace用于跟踪函数与方法的调用，此处以寻找发包函数为例搜索request，先启动APP，在电脑执行frida-trace命令。

![](./2024/03/17/从0到1的IOS APP安全测试/image_rJ12Hhict4.png)

执行完之后会在当前目录下自动生成 【__handlers__ 】目录和示例脚本，进入handlers目录下编辑相应的js文件。

![](./2024/03/17/从0到1的IOS APP安全测试/image_awq4MNTNA-.png)

通过编写frida-trace脚本可通过打印参数值查看其参数值。

![](./2024/03/17/从0到1的IOS APP安全测试/image_6gZqVEhh6-.png)

成功获得发包函数中的传参值，关于案例可以参考我的另一篇文章：https://www.freebuf.com/articles/mobile/393147.html
